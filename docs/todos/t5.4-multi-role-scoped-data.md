# T5.4 Multi-Role Scoped Data — Testing Plan

## Overview
Tests multi-role teacher switching schools via `x-school-id` header to verify all module endpoints properly isolate data by the selected school context.

**Auth Header Format:** `authorization: Bearer <token>`

---

## Phase 1 — Login/Registration Context Selection (`login-context-verification.test.ts`)

> Tests that a teacher user with roles at schools A and B gets correct roles populated in `resolvedRoles` based on `x-school-id`

- [x] **T1.1** Teacher with roles at schools A+B logging in with `x-school-id=A` → resolvedRoles contains school A role
- [x] **T1.2** Same user logging in with `x-school-id=B` → resolvedRoles contains school B role
- [x] **T1.3** GET /profile with `x-school-id=A` returns school A role context
- [x] **T1.4** GET /profile with `x-school-id=B` returns school B role context
- [x] **T1.5** GET /roles returns all roles for both schools
- [x] **T1.6** User with school-specific academic year and x-school-id=A → matched to school A role
- [x] **T1.7** User with school-wide academic year and x-school-id=A → matched to school A role
- [x] **T1.8** Global role user (`school_id` = NULL) with `x-school-id=A` → resolvedRoles = [] (no matching role)

**Phase 1 Summary:** All 8 tests passing ✅
