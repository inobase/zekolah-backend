# v1.0.3 — Backend Test Fixes (DECIMAL → Number, Knex Collation Patch)

## Tanggal
2026-07-16

## Summary
Memperbaiki **semua 4 tes yang gagal** (dari 186→190 passing) sehingga seluruh test suite mencapai **190/190 passing**. Dua root cause utama:

1. **MySQL DECIMAL kolom (`max_score`, `score`) dikembalikan sebagai string** (`"100.00"`) karena `dateStrings: true` di config database, menyebabkan assertion `expected '"100.00"' to be 100` pada assignment & grade tests.
2. **Knex 3.x `whereLike` hardcode `COLLATE utf8_bin`**, yang tidak valid untuk charset `utf8mb4` MySQL, menyebabkan HTTP 500 pada semua endpoint search (school, subject, teacher, user).

## Root Cause Analysis

### DECIMAL → String
`src/config/database.ts` memiliki konfigurasi `dateStrings: true` pada koneksi MySQL. Opsi ini membuat **semua** kolom yang dibaca MySQL2 driver dikembalikan sebagai string, bukan tipe native. Termasuk DECIMAL columns seperti `max_score` dan `score`.

```typescript
// Sebelum — dateStrings: true meng-conversi DECIMAL jadi string
connection: {
  // ...
  dateStrings: true,  // <-- ini penyebabnya
},
```

Contoh failure di test output:
```
AssertionError: expected '100.00' to be 100 // Object.is equality
```

### Knex utf8_bin Collation Bug
Knex 3.x source code di `node_modules/knex/lib/dialects/mysql/query/mysql-querycompiler.js` secara hardcode:

```javascript
whereLike(statement) {
  return `${this._columnClause(statement)} ${this._not(
    statement, 'like '
  )}${this._valueClause(statement)} COLLATE utf8_bin`;
}
```

Kolation `utf8_bin` tidak ada di charset `utf8mb4` — hanya tersedia di charset `utf8` (latin1). Akibatnya semua query search dengan `.whereLike()` gagal:

```
Error: select * from `schools` where (`name` like '%SMA%' COLLATE utf8_bin ...) 
- COLLATION 'utf8_bin' is not valid for CHARACTER SET 'utf8mb4'
```

## Perubahan

### 1. `src/config/database.ts` — Monkey-patch Knex whereLike
Menambahkan patch setelah import Knex agar `COLLATE utf8_bin` diganti `COLLATE utf8mb4_general_ci` sebelum Knex instance dibuat:

```typescript
const MySQLQueryCompiler = require('knex/lib/dialects/mysql/query/mysql-querycompiler.js');
if (MySQLQueryCompiler && MySQLQueryCompiler.prototype) {
  const originalWhereLike = MySQLQueryCompiler.prototype.whereLike;
  MySQLQueryCompiler.prototype.whereLike = function (statement: any) {
    return originalWhereLike
      .call(this, statement)
      .replace('COLLATE utf8_bin', 'COLLATE utf8mb4_general_ci');
  };
}
```

**Kenapa patch daripada fork/upgrade?**
- Fix minimal impact, hanya mengubah baris yang error.
- Tidak perlu upgrade Knex versi (bisa introduce breaking changes lain).
- Tidak perlu mengganti semua `.whereLike()` ke `.whereRaw('LIKE ...')`.
- Jika nanti Knex versi baru memperbaiki ini, patch bisa dihapus tanpa merusak logic lain.

### 2. `src/repositories/assignment.repository.ts` — Cast `max_score` ke Number
`findById()`:
```typescript
// Sebelum
return row || null

// Sesudah  
return { ...row, max_score: Number(row.max_score) } as unknown as AssignmentWithDetails
```

`findAll()`:
```typescript
// Sebelum
return query.orderBy(...)

// Sesudah
const rows = await query.orderBy(...)
return rows.map((r: any) => ({ ...r, max_score: Number(r.max_score) })) as AssignmentWithDetails[]
```

### 3. `src/repositories/grade.repository.ts` — Cast `max_score` & `score` ke Number
```typescript
// findAll
return rows.map((r: any) => ({
  ...r, max_score: Number(r.max_score), score: Number(r.score)
})) as GradeWithDetails[]

// findById
return { ...row, max_score: Number(row.max_score), score: Number(row.score) } as unknown as GradeWithDetails
```

### 4. `src/repositories/submission.repository.ts` — Cast `score` ke Number
```typescript
// findAll & findById
return {
  ...row,
  score: row.score !== null && row.score !== undefined ? Number(row.score) : null,
} as unknown as SubmissionWithDetails
```

Nilai `null` diperlakukan khusus karena `score` di submissions boleh NULL (belum di-grade).

## Trade-offs

| Trade-off | Catatan |
|-----------|---------|
| Monkey-patch Knex | Modifikasi prototype Knex internal — tidak ideal untuk production jangka panjang, tapi workaround yang aman. Jika Knex di-upgrade, patch bisa di-drop atau disesuaikan. |
| Cast di repository | Tiap repository yang mengembalikan DECIMAL fields harus cast eksplisit. Pertimbangkan membuat utility `normalizeRow()` untuk menghindari duplikasi. |
| `dateStrings: true` tetap aktif | Untuk jangka panjang pertimbangkan menonaktifkan opsi ini dan configure `typeCast` di mysql2 lebih granular agar hanya tanggal yang dikembalikan sebagai string. |
