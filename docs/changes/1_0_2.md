## 1.0.2 — RESTful CRUD Endpoints untuk Modul Akademik

### Added

- **7 route baru** untuk modul akademik dengan pola RESTful standar:

| Route | Endpoint Prefix | Deskripsi |
|-------|-----------------|-----------|
| Academic Year | `/academic-years` | Manajemen tahun ajaran per sekolah (CRUD + filter + pagination) |
| Attendance | `/attendances` | Rekam kehadiran siswa per mata pelajaran (CRUD + filter tanggal/status) |
| Assignment | `/assignments` | Tugas/PR yang dibuat guru untuk kelas tertentu (CRUD + pagination) |
| Submission | `/submissions` | Pengumpulan tugas siswa dan grading (CRUD + scoring) |
| Grade | `/grades` | Nilai siswa per mata pelajaran (CRUD + filter assessment type) |
| Teaching Assignment | `/teaching-assignments` | Penugasan guru mengajar kelas+mapel per tahun ajaran (CRUD + pagination) |

- **Pagination** — Setiap endpoint yang support list mengembalikan struktur respons konsisten:
  ```json
  {
    "data": [...],
    "pagination": {
      "page": 1,
      "limit": 20,
      "total": 100
    }
  }
  ```

- **Filtering** — Query params untuk filter di setiap route sesuai kebutuhan bisnis (misal: `student_id`, `subject_id`, `date_from`, `date_to`, `assessment_type`)

- **Validasi input** — Setiap POST/PATCH melakukan validasi required fields dan mengembalikan 400 Bad Request jika field wajib tidak ada

### Changed

- **Konfigurasi database (`src/config/database.ts`)**
  - **Switch ke Knex** sebagai ORM database (pengganti seeder manual)
  - Mendukung **MySQL** untuk production dan **SQLite in-memory** untuk testing
  - Singleton pattern dengan `getKnex()` export
  - Foreign key enforcement aktif di SQLite via `pragma('foreign_keys = ON')`

- **Struktur middleware (`src/middlewares/`)**
  - Memindahkan `errorHandler` dari `app.ts` ke file terpisah `src/middlewares/errorHandler.ts`
  - Global error handler menangani: Zod validation errors, JWT auth errors, Knex/Mysql schema errors

- **Logging (`src/utils/logger.ts`)**
  - Utility logger sederhana berbasis `console` untuk fase development
  - Method: `info()`, `warn()`, `error()`, `debug()`

- **API Response format**
  - Error responses memiliki struktur konsisten: `{ statusCode, error, message, details? }`
  - POST/PATCH responses mengembalikan entitas yang baru dibuat/diubah

### Fixed

- **Migration FK constraint errors** — Perbaikan urutan pembuatan tabel di `migrations/001_create_base_tables.ts`:
  - `students` dibuat **sebelum** `class_students` (agar FK constraint valid)
  - `teachers` dibuat **sebelum** `teaching_assignments`

### Deleted

- **Sekuler login manual** — Migrasi dari seeder manual ke Knex migration

### Architecture Notes

- Pola arsitektur yang diterapkan:
  - **Singleton** untuk Knex instance
  - **Factory** untuk Fastify app (`src/app.ts` → `buildApp()`)
  - **Decorator pattern** untuk auth (`app.decorate('authenticate', ...)`)
  - **Middleware separation** untuk error handling

## 1.0.1 — Refaktor Arsitektur

### Added
- **Type-check script** — Menambahkan script `npm run type-check` menggunakan `tsc --noEmit` untuk memastikan kode bebas error TypeScript sebelum deployment
- **Banner info host & port** — Saat dev server dijalankan (`npm run dev`), console menampilkan banner informatif berisi:
  - Host yang diaktifkan
  - Port yang digunakan
  - URL lengkap
  - Environment (development/production)
  - Versi aplikasi
  - Info API base dan health endpoint
- **Migrate script terpusat** — Membuat `scripts/migrate.js` yang menangani semua operasi database (latest, rollback, seed) dengan otomatis memuat `.env`

### Fixed
- **Error `user already decorated`** — Memperbaiki konflik dekorasi `request.user` yang disebabkan oleh deklarasi ganda dari `@fastify/passport` dan module augmentation. Menghapus依赖 `@fastify/passport` karena tidak digunakan (hanya JWT yang dipakai)
- **FK constraint error pada migration** — Memperbaiki urutan pembuatan tabel di `migrations/001_create_base_tables.ts`. Tabel `students` sekarang dibuat **sebelum** `class_students` agar foreign key constraint valid (MySQL errno 150)
- **Migration script tidak load .env** — Script migrasi sebelumnya tidak memuat `.env`, menyebabkan error "Access denied (using password: NO)". Sekarang menggunakan `require('dotenv').config()` di awal

### Changed
- **Struktur routes menggunakan decorator pattern** — Menambahkan `app.decorate('authenticate', ...)` di `src/index.ts` sehingga semua route bisa menggunakan `app.addHook('onRequest', app.authenticate)` tanpa import manual
- **Logger sederhana** — Mengganti logger kompleks dengan utility `src/utils/logger.ts` berbasis console yang lebih ringan untuk fase development
- **Type declaration terpusat** — Menggabungkan type augmentation dari `types/fastify.d.ts` dan `middlewares/authGuard.ts` ke dalam `src/index.ts` menggunakan `declare module '@fastify/jwt'` dan `declare module 'fastify'`

### Files Removed
- `src/config/passport.ts` — Tidak diperlukan, hanya JWT yang digunakan
- `src/middlewares/authGuard.ts` — Fungsi authenticate dipindah ke decorator di `src/index.ts`
- `src/types/fastify.d.ts` — Type augmentation digabung ke `src/index.ts`

### Files Added
- `scripts/migrate.js` — Script terpusat untuk `latest`, `rollback`, dan `seed`
- `src/utils/logger.ts` — Logger utility sederhana
- `scripts/` directory (baru) — Tempat script helper CLI

### Dependencies
- **Added:** `dotenv@^17.4.2` — Untuk memuat `.env` file di scripts
- **Removed:** `@fastify/passport` — Tidak digunakan, hanya JWT
- **Removed:** `@types/passport-local` — Tidak diperlukan lagi

## Migration Guide

### Dari 1.0.0 ke 1.0.1
1. Hapus manual `@fastify/passport` dan `passport-local` dari `package.json`
2. Jalankan ulang migration: `npm run migrate`
3. Pastikan `type-check` berjalan: `npm run type-check`

### Dari 1.0.1 ke 1.0.2
1. Tidak ada breaking changes pada database schema
2. Hanya penambahan route-endpoints baru
3. Middleware `errorHandler` sekarang terpisah dari `app.ts`
