# CHANGELOG v1.0.5 — Swagger Integration & Type Safety Modernization

**Date:** 2026-07-17  
**Scope:** Backend API — Swagger/OpenAPI 3.x, Zod type provider, CORS hardening, Content-Type validation

---

## Summary

This release completes the migration of all 65+ API routes to use `fastify-type-provider-zod` with full OpenAPI/Swagger documentation. Key improvements include:

- **Full Swagger integration** — all routes now have typed request/response schemas, summaries, descriptions, tags, and security definitions.
- **CORS production readiness** — configurable `CORS_ORIGIN` env var with `NODE_ENV`-aware fallback logic.
- **Content-Type enforcement** — POST/PUT/PATCH requests without `multipart` must use `application/json` or receive `415 Unsupported Media Type`.
- **Reusable OpenAPI examples** — 7 component examples for standard response patterns (login, register, pagination, errors).
- **Context header documentation** — `X-School-ID` and `X-Academic-Year-ID` headers documented on all 50+ authenticated endpoints.

## Content-Type Validator Plugin

The new `content-type-validator` plugin enforces:

- **POST/PUT/PATCH** without `multipart` → requires `Content-Type: application/json`
- **Allowed:** `multipart/form-data` (for file uploads like submissions)
- **Rejected:** any other Content-Type → `415 Unsupported Media Type` with descriptive error message

Skipped paths: `/api/v1/submissions` (file upload endpoint).

## Backward Compatibility

- **Breaking:** None. All changes are additive — schemas and middleware don't change existing API behavior.
- **New:** 415 responses for invalid Content-Type on POST/PUT/PATCH (unlikely to affect consumers using JSON clients).
- **New:** CORS behavior is stricter when `CORS_ORIGIN` is not set in production (defaults to `http://localhost:3001` instead of `*`).
